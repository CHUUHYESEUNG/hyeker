# 블로그 운영 전략

## 📊 현재 상황

### 기존 플랫폼 분산
- **티스토리**: 기술 블로그 (가볍게 쓰는 기술 블로그)
- **네이버 블로그**: 일상을 담는 공간
- **브런치**: 에세이
- **hyeker.com**: 새로 구축한 메인 블로그 (현재 더미 데이터만 존재)

### 문제점
- 콘텐츠가 여러 플랫폼에 분산되어 있음
- 독자층이 나뉘어져 있음
- 포트폴리오와 블로그의 시너지가 약함
- 브랜드 이미지가 명확하지 않음

---

## 🎯 선택한 전략: 메인 허브 + 유통 채널

### 핵심 개념
hyeker.com을 **메인 허브**로, 기존 플랫폼은 **유입 채널**로 활용

```
hyeker.com (원본, 풀 버전)
    ↓ 요약 + 링크
    ├─ 티스토리 (기술 글 요약)
    ├─ 네이버 블로그 (일상 글 요약)
    └─ 브런치 (에세이 전문 게재)
```

### 이 전략을 선택한 이유

1. **포트폴리오 시너지**: 블로그 독자가 자연스럽게 포트폴리오 확인
2. **SEO 집중**: 모든 콘텐츠의 canonical은 hyeker.com
3. **기존 자산 활용**: 티스토리/네이버의 검색 파워 활용
4. **점진적 전환**: 독자들이 자연스럽게 hyeker.com으로 이동
5. **유연성**: 나중에 완전 통합으로 전환 가능

## 📝 콘텐츠 운영 방식

### 1. 새 글 발행 프로세스

**Step 1: hyeker.com에 풀 버전 작성** (1차 발행)
- 코드 예제, 이미지, 상세 설명 모두 포함
- 관련 포트폴리오/프로젝트 링크 추가
- SEO 최적화된 메타데이터 작성

**Step 2: 기술 글인 경우 → 티스토리 요약 게재**
- 핵심 내용 요약 (30% 정도)
- "전문 보기: https://hyeker.com/blog/xxx" 링크 추가
- 예시:
  ```
  Next.js 15의 새로운 기능을 알아봤습니다.

  [핵심 요약 3~4 문단]

  ---
  📖 전문 읽기: https://hyeker.com/blog/nextjs-15-features
  제 개인 블로그에서 코드 예제와 함께 더 자세한 내용을 확인하세요!
  ```

**Step 3: 일상 글인 경우 → 네이버 블로그 요약 게재**
- 가볍게 요약
- hyeker.com 링크 추가

**Step 4: 에세이인 경우 → 브런치**
- 전문 게재 (브런치 특성상 긴 글 선호)
- 또는 hyeker.com 링크만

### 2. 글감 분류

#### hyeker.com 독점 게재
- 프로젝트 회고 (온유, 매듭, 캐치픽 등)
- 기술 심화 글 (아키텍처, 성능 최적화)
- 포트폴리오 케이스 스터디
- 연간 회고

#### 전체 플랫폼 배포 (요약 + 링크)
- 개발 튜토리얼
- 기술 트렌드 분석
- 도구/라이브러리 리뷰

#### 브런치 전문
- 커리어 전환 스토리
- 개발자 일상 에세이
- 임신/육아 + 개발 병행기

#### 네이버 전문
- 임신 준비 일상
- 워라밸 팁
- 서평, 제품 리뷰

## 🚀 실행 로드맵

### Phase 1: 기반 구축 (1~2주)

**Week 1: 초기 콘텐츠 생성**
- [ ] hyeker.com 블로그에 샘플 글 3~5개 작성
  - 기술 글 2개
  - 회고/에세이 2개
  - 프로젝트 소개 1개
- [ ] RSS 피드 생성 (선택)
- [ ] 뉴스레터 구독 기능 추가 (선택)

**Week 2: 기존 플랫폼 안내**
- [ ] 티스토리에 "새 블로그 안내" 공지글
- [ ] 네이버 블로그에 "메인 블로그 이전" 공지
- [ ] 브런치 프로필에 hyeker.com 링크 추가

### Phase 2: 콘텐츠 마이그레이션 (1개월)

**선별 기준**:
- 조회수 상위 10개 글
- 검색 유입이 많은 글
- 포트폴리오와 연관된 글
- 에버그린 콘텐츠 (시간이 지나도 유효한 글)

**마이그레이션 프로세스**:
1. 기존 글 → hyeker.com으로 재작성 (개선해서)
2. 기존 플랫폼 글 끝에 추가:
   ```
   📍 이 글의 최신 버전은 [제 개인 블로그](https://hyeker.com/blog/xxx)에서 확인하실 수 있습니다.
   더 나은 읽기 경험과 관련 콘텐츠를 제공합니다.
   ```

### Phase 3: 지속 운영 (계속)

**발행 주기**:
- 주 1회 발행 목표
- 짧은 글도 OK (500자도 괜찮음)
- 완벽주의 버리기

**루틴**:
1. 주말에 글 작성
2. 월요일 hyeker.com 발행
3. 화요일 다른 플랫폼 배포

- **발행**: 브런치 + hyeker.com 전문
- **배포**: 티스토리/네이버 요약 + 링크
- **목적**: 커리어 전환에 관심 있는 독자 타겟

- **기대 효과**: 개발자 독자 유입

## ⚠️ 주의사항

### SEO 중복 컨텐츠 방지
- 다른 플랫폼은 반드시 **요약만** 게재
- canonical URL은 항상 hyeker.com
- 전문 링크 명확히 표시

### 독자 이탈 최소화
- 기존 플랫폼 글 **삭제하지 말기**
- 공지는 친절하게, **강요하지 않기**
- "더 나은 경험" 강조

### 지속 가능한 운영
- 완벽주의 버리기
- 짧은 글도 OK
- 주 1회 발행 목표로 시작
- 번아웃 방지
